What Is NestJS?

  • NodeJS framework
  • Provides an out-of-the-box strict architecture
  • Highly integrated with TS
  • Angular-like architecture divided into modules
  • Good CLI

Why should we use this framework?

  • Offers a lot of new functionality in comparison Express JS
  • Operates on a clear architecture that helps divide applications into microservices using simple components like modules, controllers, and providers.

Advantages of the NestJS

  • Leverages TypeScript
  • Powerful CLI
  • Contains abuild-in dependency injection
  • Lazy Loading modules
  • Supports a lot of Nest-specific modules
  • Easy unit-testing
  • Provides detailed documentation

Main concepts of NestJS

  • Modules
  • Providers
  • Controllers

Modules

Modules are building blocks of application, they help to organize code.

Providers

Are the fundamental concept. They can be injected as a dependency. Objects in NestJS can create various relationships with each other.

Controllers

Controllers are responsible for handling incoming requests and returning responses to the client.

How to use NestJS?

Installation

  • Requires Node JS version 12+
  • To install Nest CLI globally use this command:
    npm i -g @nestjs/cli

Startting new project

  • Use this command to start new project: 
    nest new project-name
